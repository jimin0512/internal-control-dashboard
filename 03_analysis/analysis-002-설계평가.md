# 설계평가 미흡 프로세스 분석

## 관련 질문

[[q-002-설계평가]]

## 사용 데이터

- [[schema-ic_design_assessment]]
- [[schema-ic_control_master]]

## 분석 방법

`ic_design_assessment.csv`(180건)의 설계평가결과 컬럼 값별 건수를 집계하고, 통제ID를 기준으로 `ic_control_master.csv`와 연결해 프로세스명을 매칭했다. 프로세스별로 설계평가결과="적정" 건수와 전체 건수를 집계해 설계 적정률을 계산했다. Key여부와 결합해 Key 통제와 비Key 통제의 설계 적정률도 각각 계산했다.

**174 / 180 (96.7%)**

설계평가결과 분포: 적정 174/180(96.7%), 개선필요 5/180(2.8%), 미적용 1/180(0.6%)

### 개선필요사유 (적정이 아닌 6건)

| 통제ID | 프로세스 | 설계평가결과 | 개선필요사유 |
|---|---|---|---|
| C0019 | 구매 | 개선필요 | 시스템 변경 후 통제절차가 갱신되지 않음 |
| C0029 | 구매 | 미적용 | 증빙 보관 기준이 정의되지 않음 |
| C0053 | 재고 | 개선필요 | 시스템 변경 후 통제절차가 갱신되지 않음 |
| C0071 | 자금 | 개선필요 | 통제기술서의 수행절차가 구체적이지 않음 |
| C0141 | 전산일반통제 | 개선필요 | 시스템 변경 후 통제절차가 갱신되지 않음 |
| C0148 | 전산일반통제 | 개선필요 | 승인권자와 수행자의 역할 구분이 불명확함 |

### Key 통제 / 비Key 통제 설계 적정률

- Key 통제: 95 / 99 (96.0%)
- 비Key 통제: 79 / 81 (97.5%)
- 적정이 아닌 6건 중 Key 통제는 C0053, C0071, C0141, C0148 (4건), 비Key 통제는 C0019, C0029 (2건)이다.

## 발견사항

- 설계평가결과가 "적정"이 아닌 6건은 12개 프로세스 중 구매·재고·자금·전산일반통제 4개 프로세스에서만 발생했다.
- 개선필요사유 6건 중 3건("C0019", "C0053", "C0141")이 "시스템 변경 후 통제절차가 갱신되지 않음"으로 동일한 문구로 기재되어 있다.
- Key 통제 설계 적정률(96.0%)이 비Key 통제 설계 적정률(97.5%)보다 낮은 수치로 나타난다.
- RCM반영여부 컬럼은 C0029(미적용)와 C0053(개선필요) 2건만 "N"이다.
