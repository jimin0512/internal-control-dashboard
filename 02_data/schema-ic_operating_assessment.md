# ic_operating_assessment 스키마 노트

차수별(1차/2차/3차) 통제 운영평가 결과를 담은 데이터다.

## 전체 행 수

540건 (통제 180건 × 3개 차수)

## 기본키와 연결키

- 기본키: 운영평가ID (예: OE-C0001-1차)
- 연결키: 통제ID → [[schema-ic_control_master]]
- 하위 연결: 운영평가ID → [[schema-ic_deficiency]]

## 필드 목록 및 의미

| 필드 | 의미 |
|---|---|
| 운영평가ID | 운영평가 고유 식별자. `OE-통제ID-차수` 형식 |
| 통제ID | 대상 통제 ID |
| 평가연도 | 평가 연도 (확인된 값: 2026) |
| 평가차수 | 1차/2차/3차 각 180건씩 |
| 평가기준일 | 평가 기준일 (1차 2026-04-30, 2차 2026-08-31, 3차 2026-12-31로 고정 패턴 확인) |
| 표본수 | 평가 표본 수 |
| 예외건수 | 표본 중 예외(오류) 발견 건수 |
| 증빙상태 | 완료 518건, 미제출 13건, 불완전 9건 |
| 기한내제출여부 | 기한 내 제출 여부. Y 467건, N 73건 |
| 운영평가결과 | 적정 505건, 미흡 35건 |
| 평가자 | 평가 수행자 (확인된 값: 내부회계팀) |
| 검토완료여부 | 검토 완료 여부 (확인된 값: 전 건 Y) |
| 비고 | 비고 |

## 데이터 간 연결 관계

- `ic_control_master.통제ID` → `ic_operating_assessment.통제ID`
- `ic_operating_assessment.운영평가ID` → `ic_deficiency.운영평가ID`

## 분석 시 주의사항

- 검토완료여부가 전 건 Y로 확인되어, 현재 raw 기준으로는 검토 미완료 현황 분석은 결과값이 0건이 된다.
- 예외건수가 0보다 크다고 반드시 미흡으로 이어지는 것은 아니므로, 적정/미흡 판단은 운영평가결과 컬럼을 기준으로 하고 예외율은 별도 참고지표로 다룬다.
- 기한내제출여부 N이 73건으로 상당수 존재하나 운영평가결과와 반드시 연동되지는 않으므로 구분해서 분석한다.
- 통제당 3개 차수가 모두 존재하므로 차수별 개선·악화 추이를 통제 단위로 비교할 수 있다.

## 자동보고서에서 활용 가능한 항목

- 평가차수별 운영 적정률, 표본수 대비 예외율, 증빙 미완료(미제출·불완전) 건수, 기한 내 미제출 현황 — 이 파일 단독 계산 가능
- 프로세스별/Key 통제별 운영 적정률: [[schema-ic_control_master]] 연결 필요 (연결키: 통제ID)
- 운영평가 미흡 건의 미비점 전환율: [[schema-ic_deficiency]] 연결 필요 (연결키: 운영평가ID)

## 보고서에서 사용될 섹션

- 4. 운영평가 결과

## KPI 연결 구조

```

→ 미비점 수 및 발생률 (ic_deficiency.csv, 연결키: 운영평가ID)
```

- (기술적) 증빙상태가 미제출 또는 불완전인 건은 몇 건이며 어떤 통제에 집중되는가?
- (진단적) 기한 내 미제출 건이 운영평가결과 미흡과 실제로 상관관계가 있는가?
- (진단적) 표본수가 적은(예: 1건) 평가에서 미흡 판정 비율이 다른가?
- (예측적) 1차에서 미흡했던 통제가 2·3차에도 반복해서 미흡 판정을 받는 경향이 있는가?
- (예측적) 증빙 미제출·불완전 건이 다음 차수에도 재발할 가능성이 있는 통제는 무엇인가?
- (처방적) 반복적으로 미흡 판정을 받는 통제에 대해 어떤 우선 조치가 필요한가?
- (처방적) 기한 내 미제출 비율이 높은 통제수행부서에 어떤 프로세스 개선이 필요한가?
