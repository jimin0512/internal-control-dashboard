# ic_deficiency 스키마 노트

운영평가 및 설계평가에서 발견된 미비점을 담은 데이터다.

## 전체 행 수

41건

## 기본키와 연결키

- 기본키: 미비점ID (예: I0001)
- 연결키: 운영평가ID → [[schema-ic_operating_assessment]] (설계평가 단계에서 발견된 6건은 공란)
- 연결키: 통제ID → [[schema-ic_control_master]]
- 하위 연결: 미비점ID → [[schema-ic_remediation]]

## 데이터 간 연결 관계

- `ic_operating_assessment.운영평가ID` → `ic_deficiency.운영평가ID` (41건 중 35건, 나머지 6건은 설계평가 발견분으로 공란)
- `ic_control_master.통제ID` → `ic_deficiency.통제ID`
- `ic_deficiency.미비점ID` → `ic_remediation.미비점ID`
- 설계평가 단계 발견 6건(통제ID: C0019, C0029, C0053, C0071, C0141, C0148)은 [[schema-ic_design_assessment]]에서 설계평가결과가 개선필요 또는 미적용으로 확인된 통제와 일치한다.

## 분석 시 주의사항

- 운영평가ID가 공란인 6건은 결측이 아니라 "설계평가 단계 발견"을 의미하므로, 운영평가 기준 미비점 집계 시 이 6건을 별도로 구분해야 한다.
- 중요도 상은 4건뿐으로 표본이 매우 작다. 비율 분석 시 분모가 작다는 한계를 함께 명시한다.
- 반복미비점여부 Y는 6건으로, 이 통제들은 우선 관리 대상으로 볼 근거가 있다.

## 자동보고서에서 활용 가능한 항목

- 프로세스별·중요도별·미비점유형별 미비점 수, 반복 미비점 목록, 근본원인별 미비점 분포, 개선담당부서별 미비점 수, 미종결(개선중·지연) 미비점 현황 — 이 파일 단독 계산 가능
- 미비점 발생률(운영평가 건수 대비): [[schema-ic_operating_assessment]] 연결 필요 (연결키: 운영평가ID/통제ID)
- 미비점별 개선조치 완료 여부 및 지연 현황: [[schema-ic_remediation]] 연결 필요 (연결키: 미비점ID)

