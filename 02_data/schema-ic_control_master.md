# ic_control_master 스키마 노트

내부회계 통제 및 관련 위험에 대한 마스터 데이터다.

## 전체 행 수

180건

## 기본키와 연결키

- 기본키: 통제ID
- 연결키: 프로세스ID → [[schema-ic_process_master]]
- 하위 연결: 통제ID → [[schema-ic_design_assessment]], [[schema-ic_operating_assessment]], [[schema-ic_deficiency]], [[schema-ic_remediation]]

## 데이터 간 연결 관계

- `ic_process_master.프로세스ID` → `ic_control_master.프로세스ID`
- `ic_control_master.통제ID` → `ic_design_assessment.통제ID` → `ic_operating_assessment.통제ID` → `ic_deficiency.통제ID` → `ic_remediation.통제ID`

## 분석 시 주의사항

- 통제책임부서와 통제수행부서가 다른 경우가 다수 있어(예: 통제책임 영업관리팀 / 수행 재무회계팀), 부서별 집계 시 두 컬럼을 혼동하지 않는다.
- 사용중여부가 전 건 Y이므로 현재 raw 데이터에는 사용 중단된 통제가 없다.
- 신규통제여부 Y는 16건으로 소수이며, 신규통제의 설계·운영평가 결과는 별도로 주의 깊게 볼 필요가 있다.

## 자동보고서에서 활용 가능한 항목

- 프로세스별·부서별 통제 수 및 Key 통제 비율 (이 파일 단독 계산 가능)
- 통제유형/통제방법/수행주기별 분포 (단독 계산 가능)
- 신규통제 현황 (단독 계산 가능)
- Key 통제 설계 적정률: [[schema-ic_design_assessment]] 연결 필요 (연결키: 통제ID)
- Key 통제 운영 적정률: [[schema-ic_operating_assessment]] 연결 필요 (연결키: 통제ID)
- 통제유형·통제방법별 미비점 발생률: [[schema-ic_deficiency]] 연결 필요

## 보고서에서 사용될 섹션

- 2. 프로세스 및 통제 현황

## KPI 연결 구조

```
프로세스별 통제 수 (본 파일, 연결키: 프로세스ID)
→ Key 통제 비율 (본 파일, Key여부 컬럼)
→ 설계 적정률 (ic_design_assessment.csv, 연결키: 통제ID)
→ 운영 적정률 (ic_operating_assessment.csv, 연결키: 통제ID)
→ 미비점 수 및 발생률 (ic_deficiency.csv, 연결키: 통제ID)
→ 반복 미비점 (ic_deficiency.csv, 반복미비점여부 컬럼)
→ 개선조치 완료율 (ic_remediation.csv, 연결키: 통제ID)
```

## 관련 질문 후보

- (기술적) 프로세스별 Key 통제 수와 비율은 어떻게 되는가?
- (기술적) 통제책임부서와 통제수행부서가 다른 통제는 몇 건이며 어느 부서 조합이 가장 많은가?
- (진단적) 특정 통제방법(예: 수기통제)에서 미비점 발생률이 더 높은가?
- (진단적) Key여부가 Y인 통제와 N인 통제 간 운영 적정률에 차이가 있는가?
- (예측적) 신규통제 16건 중 설계평가 미흡 판정을 받은 통제가 있는가, 향후 추가로 미흡 판정 가능성이 있는 통제는 무엇인가?
- (예측적) 통제수행부서가 IT팀인 통제들이 향후 미비점 재발 위험이 더 높은가?
- (처방적) Key 통제 중 수기통제 비중이 높은 프로세스는 자동화 전환 우선순위 대상인가?
- (처방적) 통제책임부서와 통제수행부서가 불일치하는 통제에 대해 책임 소재를 재정비해야 하는가?
