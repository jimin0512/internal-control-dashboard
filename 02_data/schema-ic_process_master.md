# ic_process_master 스키마 노트

내부회계관리제도의 프로세스 기준정보를 담은 마스터 데이터다.

## 전체 행 수

12건

## 기본키와 연결키

- 기본키: 프로세스ID
- 하위 연결: [[schema-ic_control_master]]의 프로세스ID와 1:N으로 연결된다.

## 필드 목록 및 의미

| 필드 | 의미 |
|---|---|
| 프로세스ID | 프로세스 고유 식별자 (예: P01) |
| 프로세스명 | 프로세스 이름 (예: 매출, 구매, 재고 등) |
| 주관부서 | 해당 프로세스를 주관하는 부서 |
| 구분 | 프로세스 성격 구분값. 확인된 값: 재무, 운영, IT, ELC |
| 중요도 | 프로세스 중요도. 확인된 값: 상, 중, 하 |
| 평가주기 | 평가 수행 주기 (확인된 값: 연 3회) |
| 비고 | 비고. 대부분 공란 |

## 데이터 간 연결 관계

`ic_process_master.프로세스ID` → `ic_control_master.프로세스ID`

이 프로세스ID를 시작점으로 통제 → 설계평가 → 운영평가 → 미비점 → 개선조치까지 전체 체인을 추적할 수 있다.

## 분석 시 주의사항

- 전체 12건으로 표본이 매우 작다. 프로세스별 비교 시 건수 차이로 인한 왜곡에 유의한다.
- 구분(재무/운영/IT/ELC)이 실제로 4가지뿐인지는 12건 전수를 확인한 결과다.
- 비고 컬럼은 대부분 비어 있어 분석에 활용하기 어렵다.

## 자동보고서에서 활용 가능한 항목

- 중요도 상 프로세스 목록 (이 파일 단독 계산 가능)
- 프로세스별 통제 수: [[schema-ic_control_master]] 연결 필요 (연결키: 프로세스ID)
- 프로세스별 Key 통제 비율: [[schema-ic_control_master]] 연결 필요
- 프로세스별 설계·운영 적정률: [[schema-ic_design_assessment]], [[schema-ic_operating_assessment]] 연결 필요
- 프로세스별 미비점 수: [[schema-ic_deficiency]] 연결 필요

## 보고서에서 사용될 섹션

- 2. 프로세스 및 통제 현황

## KPI 연결 구조

이 노트는 KPI 흐름의 출발점인 "프로세스" 축 역할을 한다.

```
프로세스별 통제 수 (본 파일 + ic_control_master.csv, 연결키: 프로세스ID)
→ Key 통제 비율 (ic_control_master.csv, 연결키: 프로세스ID)
→ 설계 적정률 (ic_design_assessment.csv, 연결키: 통제ID)
→ 운영 적정률 (ic_operating_assessment.csv, 연결키: 통제ID)
→ 미비점 수 및 발생률 (ic_deficiency.csv, 연결키: 통제ID/프로세스명)
```

## 관련 질문 후보

- (기술적) 프로세스별 통제 수는 몇 개이며 어느 프로세스가 가장 많은 통제를 보유하는가?
- (기술적) 중요도 상으로 분류된 프로세스는 몇 개이며 어떤 프로세스인가?
- (진단적) 특정 프로세스(예: 재무 구분)에 미비점이 집중되는 이유는 무엇인가?
- (진단적) 평가주기가 모두 "연 3회"로 동일한데도 프로세스별 운영 적정률 차이가 나는 이유는 무엇인가?
- (예측적) 중요도 상 프로세스 중 향후 미비점 재발 위험이 높은 프로세스는 어디인가?
- (예측적) Key 통제 비율이 낮은 프로세스는 향후 설계평가에서 개선필요 판정을 받을 위험이 더 큰가?
- (처방적) 중요도 상 프로세스에 대한 평가 우선순위를 어떻게 조정해야 하는가?
- (처방적) IT 구분 프로세스(전산일반통제)에 대한 통제 강화가 필요한가?
