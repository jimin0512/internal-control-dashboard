# ic_design_assessment 스키마 노트

통제별 설계평가 결과를 담은 데이터다.

## 전체 행 수

180건 (통제 180건 각각 1건씩, 1:1 관계로 보인다)

## 기본키와 연결키

- 연결키: 통제ID → [[schema-ic_control_master]]
- 이 파일 자체는 명시적 기본키 컬럼이 없으며, 통제ID가 사실상 유일하게 식별한다(2026년 평가연도 기준 통제당 1건).

## 필드 목록 및 의미

| 필드 | 의미 |
|---|---|
| 통제ID | 통제 고유 식별자 |
| 평가연도 | 설계평가 수행 연도 (확인된 값: 2026) |
| 설계평가일 | 설계평가 수행일 |
| 설계평가결과 | 적정 174건, 개선필요 5건, 미적용 1건 |
| 개선필요사유 | 설계평가결과가 개선필요/미적용인 경우의 사유. 적정 건은 공란 |
| 평가자 | 설계평가 수행자 (확인된 값: 내부회계팀) |
| 검토자 | 평가 결과 검토자 (확인된 값: CFO) |
| RCM반영여부 | RCM(통제기술서) 반영 여부. Y 178건, N 2건 |
| 비고 | 비고 (확인된 값 없음, 공란) |

## 데이터 간 연결 관계

`ic_control_master.통제ID` → `ic_design_assessment.통제ID`

설계평가결과가 개선필요/미적용인 통제는 [[schema-ic_deficiency]]에서 발견차수="설계평가"로 표시된 미비점과 연결된다(운영평가ID는 공란).

## 분석 시 주의사항

- 현재 raw에는 2026년 평가연도 1개 차수만 존재해 연도별 추이 비교는 불가능하다.
- RCM반영여부 N 2건은 C0029(설계평가결과 미적용)와 C0053(설계평가결과 개선필요)이다.
- 개선필요사유는 적정 건에서는 공란이므로 결측이 아니라 정상값임에 유의한다.

## 자동보고서에서 활용 가능한 항목

- 전체 설계 적정률(174건/180건), 설계 미흡 통제 목록 및 개선필요사유, RCM 미반영 통제 현황(2건) — 이 파일 단독 계산 가능
- Key 통제 설계 적정률: [[schema-ic_control_master]] 연결 필요 (연결키: 통제ID, Key여부)
- 설계 미흡 통제의 이후 운영평가 결과: [[schema-ic_operating_assessment]] 연결 필요
- 설계 미흡 통제와 미비점의 연결: [[schema-ic_deficiency]] 연결 필요 (발견차수="설계평가")

## 보고서에서 사용될 섹션

- 3. 설계평가 결과

## KPI 연결 구조

```
Key 통제 비율 (ic_control_master.csv, 연결키: 통제ID)
→ 설계 적정률 (본 파일, 설계평가결과 컬럼)
→ 운영 적정률 (ic_operating_assessment.csv, 연결키: 통제ID)
```

설계평가결과가 개선필요/미적용인 6건은 미비점 발생 경로 중 하나로, [[schema-ic_deficiency]](발견차수="설계평가")로 직접 이어진다.

## 관련 질문 후보

- (기술적) 전체 설계 적정률과 Key 통제 설계 적정률은 각각 몇 %인가?
- (기술적) 설계평가결과가 개선필요/미적용인 6개 통제는 무엇이며 어느 프로세스에 속하는가?
- (진단적) 설계 미흡 통제들의 개선필요사유에 공통 패턴이 있는가?
- (진단적) RCM 미반영 통제 2건이 설계평가결과와 어떤 관계가 있는가?
- (예측적) RCM 미반영 통제가 향후 운영평가에서도 미흡으로 이어질 가능성이 있는가?
- (예측적) 설계평가결과 개선필요/미적용 통제 중 아직 개선조치가 완료되지 않은 통제는 무엇인가?
- (처방적) 설계 미흡 통제에 대한 개선 우선순위는 어떻게 정해야 하는가?
- (처방적) RCM 미반영 통제 2건은 어떤 절차로 즉시 반영해야 하는가?
